Ⅰ. 제안개요 초안

1) 제안 배경과 문제 정의

  • 본 사업은 김포·김해·제주·대구·청주 5개 공항을 대상으로 국제선 수속구간별 소요시간과 혼잡도를 안내하는 서비스를 구축하는 과제다.
  • 제안의 출발점은 공항별 운영환경이 서로 다르며, 단일 기준의 일괄 적용이 아니라 공항별 validation logic이 필요하다는 점이다.
  • 따라서 본 제안서는 "RFP-first spine + proof layer"를 기본 구조로 두고, 평가항목 정렬과 증거 추적성을 함께 확보한다.

2) 제안의 핵심 메시지

  • 핵심 메시지는 국제선 수속단계별 실시간 소요시간, 혼잡도, 예측값을 하나의 서비스 체계로 묶어 제공한다는 것이다. (S-01)
  • 제안 구조는 RFP 요구를 먼저 정렬한 뒤, 각 주장마다 근거와 적용 범위를 붙이는 proof layer 방식으로 설계한다. (S-02)
  • 5개 공항은 모두 별도 validation logic을 가지며, 김해공항 LiDAR는 공통 baseline이 아니라 제한적 reference proof anchor로만 사용한다. (S-03, S-04)
  • 데이터·보안·품질·테스트는 초기 설계 단계에서 함께 잠그고, 약한 근거는 공통 메시지로 승격하지 않는다. (S-05)

3) 제안 범위

  • 본문은 RFP 권장 목차에 맞춰 전략, 기술, 품질, 프로젝트 관리, 프로젝트 지원으로 이어지는 구조를 따른다.
  • 본 초안은 제안사 일반현황과 기타 항목을 제외하고, 실제 기술 제안의 본체가 되는 장만 먼저 작성한다.
  • 모든 주장과 문장은 Stage 1~2.5에서 고정한 claim/evidence ledger와 대응해야 한다.

4) 차별화 포인트

  • 공항별 운영 차이를 무시하지 않고, 공항별 validation card를 통해 각 공항의 데이터 가용성과 예외 규칙을 분리해서 설명한다.
  • 김해공항의 LiDAR 실적은 단순 홍보 문구가 아니라 검증 방법론의 anchor로 제한적으로 사용한다.
  • 성능 수치는 임의로 제시하지 않고, 검증 체계와 운영 적합성 중심의 서술로 신뢰도를 만든다.

5) 본문 작성 원칙

  • 승인된 claim만 사용한다.
  • Hold / Quarantine claim은 본문 공통 메시지에 넣지 않는다.
  • 본문과 발표자료는 동일한 ledger를 공유한다.
  • 익명평가, 분량, 보안 제약은 작성 초기부터 반영한다.
